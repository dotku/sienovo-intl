### 3.2 VxWorks 5.x/6.x

VxWorks 采用模块化的 **Wind River** 核心，同样适用于高可靠性场景。针对 LS1043A 的适配建议如下：

1. **BSP（Board Support Package）**  
   - NXP 官方提供 **LS1043A BSP**，包含启动代码、设备驱动和示例工程。  
   - 在 BSP 中启用 **PCIe**、**SATA** 与 **USB** 驱动，以充分利用平台的扩展能力。

2. **驱动移植**  
   - 对于 FPGA 自定义外设，可使用 **VxWorks I/O 系统**（`ioLib`）实现统一的文件句柄访问。  
   - 示例代码（读取 FPGA 状态寄存器）：
     ```c
     int fd = open("/dev/fpga0", O_RDONLY);
     if (fd < 0) {
         perror("open");
         return -1;
     }
     uint32_t status;
     read(fd, &status, sizeof(status));
     close(fd);
     ```

3. **网络协议栈**  
   - VxWorks 提供 **Wind River VxWorks TCP/IP Stack**，支持 **IPsec**、**VLAN** 与 **QoS**。  
   - 在多网口场景下，可通过 **netTask** 将不同业务流分配到独立的网口，实现链路冗余。

4. **安全特性**  
   - 利用 **Secure Boot** 与 **Trusted Execution Environment (TEE)**，确保固件在上电时完整性校验。  
   - FPGA 可配合 **Xilinx Secure Bitstream**，防止未授权的位流加载。

## 4. 开发流程示例

下面给出一个从硬件原型到系统上线的典型流程，帮助工程师快速落地 LS1043A + FPGA 方案。

| 阶段 | 关键任务 | 参考工具 |
|------|----------|----------|
| **硬件选型** | 确认 LS1043A 与所需 FPGA（如 Xilinx Zynq UltraScale+）的兼容性 | NXP/FPGA 官方手册 |
| **原理图设计** | 完成电源、时钟、PCIe、SATA、USB 的布局布线 | Altium Designer / KiCad |
| **固件编写** | 使用 Vivado/Quartus 生成 FPGA 位流，编写 U‑Boot 启动脚本 | Vivado、Quartus、U‑Boot |
| **系统移植** | 移植 QNX 或 VxWorks BSP，调试设备树、驱动 | QNX Momentics、Wind River Workbench |
| **功能验证** | 通过 **iperf**、**ping**、**modbus** 等工具验证网络、协议性能 | iperf、modbus‑cli |
| **安全加固** | 启用 Secure Boot、TPM、FPGA 加密位流 | OpenSSL、TPM2‑Tools |
| **量产准备** | 编写生产测试脚本，完成 OTA/固件升级方案 | Python、Ansible |

### 4.1 示例：在 QNX 上实现 Profibus 数据采集

1. **FPGA 位流**：在 Vivado 中实现 Profibus 接收器，将采集到的字节写入内部 RAM。  
2. **设备树**：在 `ls1043a.dts` 中添加 Profibus 设备节点，指向 FPGA 的寄存器基地址。  
3. **用户空间程序**：使用 QNX 的 **POSIX I/O** 打开 `/dev/profibus0`，循环读取并解析数据。  
4. **实时调度**：将该采集线程设置为 **SCHED_FIFO**，优先级 80，绑定至 CPU0，确保毫秒级响应。
