## 引言

随着国产化进程的加速，国产 CPU、PCIe 交换芯片以及 GPU 扩展卡的组合正在成为高性能计算与人工智能服务器的关键技术路径。本文围绕 **全国产龙芯 + PCIe 4.0/5.0 Switch + GPU 扩展卡** 的 **NVMe 混合直连背板** 方案展开，详细介绍硬件架构、关键功能特性以及实际部署时的注意事项，帮助读者快速了解该方案在边缘 AI 与工业计算场景下的应用价值。

---

## 方案概览

该解决方案由众星微提供，核心组成如下：

| 组件 | 说明 |
|------|------|
| **CPU** | 龙芯 3A3000 处理器，4 核，主频 1.2‑1.5 GHz |
| **PCIe Switch** | 支持 PCIe 4.0/5.0，提供多路高速直连通道 |
| **GPU 扩展卡** | 独立显卡，显存 1 GB，兼容集成显卡 |
| **NVMe 直连背板** | 实现 NVMe SSD 与 CPU/GPU 的低延迟直连 |
| **BMC (I2C)** | 通过 I2C 接口实现底层管理与监控 |

整体方案实现了 **硬盘热插拔、灯态指示、SPGIO 报错、分时启动、风扇温度控制** 等企业级功能，兼容国产操作系统（中标麒麟、深之度、普华等）以及国产昆仑固件。

## 硬件细节

### 1. 龙芯服务器

![](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/142145056/02_edaad37705db43d496ffbe4a9d3b1659.png)

- **CPU**：龙芯 3A3000 为国产 LoongArch 架构的四核处理器，采用 28 nm 工艺，功耗控制在 100 W 以下，适合边缘计算对功耗的严格要求。  
- **内存**：DDR3，最大支持 16 GB，满足大多数 AI 推理模型的内存需求。  
- **存储**：2.5 寸/3.5 寸 SATA 硬盘，容量不小于 500 GB，最大支持 4 TB。通过 PCIe Switch 与 NVMe 直连背板结合，可实现 SATA 与 NVMe 双通道混合存储。  
- **光驱**：内置 DVD 刻录光驱，方便离线系统镜像的写入与恢复。  
- **显示**：标配独立显卡，显存 1 GB，支持集成显卡，满足可视化监控与轻量级图形计算需求。  
- **整机规格**：重量 ≤ 9 kg（标准配置），最大功耗 ≤ 100 W（标准配置），便于在机架或机箱内部灵活部署。  

### 2. PCIe 4.0/5.0 Switch

PCIe Switch 采用最新的 4.0/5.0 规范，提供 **至少 16 条** 8 GT/s（4.0）或 **32 GT/s**（5.0）通道。其优势包括：

- **带宽聚合**：多条通道聚合后可为 GPU 扩展卡提供超过 64 GB/s 的上行带宽，满足高分辨率视频处理与深度学习训练的需求。  
- **低延迟直连**：NVMe SSD 通过 Switch 直接映射到 CPU 的 PCIe 根复合体，实现 **< 10 µs** 的 I/O 延迟。  
- **热插拔支持**：Switch 内部具备热拔插检测逻辑，配合硬盘热插拔功能，实现无中断的存储扩容。

### 3. GPU 扩展卡

独立显卡采用国产 GPU 核心（如华为昇腾、比特大陆自研），显存 1 GB，支持 OpenCL 与 Vulkan。主要用于：

- **AI 推理**：在边缘节点实时完成图像分类、目标检测等任务。  
- **工业可视化**：提供高帧率的机器视觉显示，配合显示器实现现场监控。  

### 4. NVMe 混合直连背板

背板将 **NVMe SSD** 与 **SATA 硬盘** 同时挂载在同一 PCIe Switch 上，形成 **混合直连** 架构。优势如下：

- **统一管理**：通过统一的驱动层实现 SATA 与 NVMe 的统一调度，降低系统软件复杂度。  
- **弹性存储**：热插拔硬盘可在运行时快速加入或移除，适用于现场维护。  
- **高可靠性**：灯态指示与 SPGIO 报错功能帮助运维人员快速定位故障硬盘。

![](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/142145056/01_1a7b89b831b44f14bde73f253faa1dd4.png)

### 1. 硬盘热插拔

- **实现方式**：PCIe Switch 检测到 SATA 端口的电源变化后，向 BMC 发送热插拔事件。  
- **运维价值**：现场更换硬盘无需关机，极大提升系统可用性。

### 2. 灯态指示

- **上电灯**：硬盘通电后 LED 亮起，提示硬盘已准备就绪。  
- **读写灯**：读写操作期间灯光闪烁，帮助快速判断磁盘 I/O 状态。  
- **报错灯**：硬盘出现 I/O 错误或 SMART 预警时灯光变红，配合 SPGIO 报错信息定位故障。

### 3. SPGIO 硬盘报错

SPGIO（Serial Peripheral GPIO）提供硬盘错误的低层信号输出，BMC 可以捕获并上报至管理平台，实现 **硬件级别的故障预警**。

### 4. 硬盘分时启动

系统支持 **分时段启动** 硬盘，例如在夜间低负载时关闭部分磁盘以降低功耗，白天再自动恢复。

### 5. 风扇温度控制

- **温度检测**：通过 I2C 读取 CPU、GPU、NVMe SSD 的温度传感器。  
- **自适应调速**：风扇转速根据温度曲线自动调节，保持系统在 **≤ 85 °C** 的安全范围内。

### 6. I2C（BMC）

BMC（Baseboard Management Controller）通过 I2C 总线实现对电源、风扇、硬盘状态的统一监控。运维平台可通过 IPMI 或 Redfish 接口远程查询、控制这些硬件资源。

## 软件生态

### 操作系统兼容性

- **中标麒麟**、**深之度**、**普华** 等国产 Linux 发行版均已通过认证，提供完整的驱动栈与内核补丁。  
- **昆仑固件**：基于开源 UEFI 实现，支持安全启动（Secure Boot）与固件升级。

### 驱动与工具

- **PCIe Switch 驱动**：已集成进 Linux kernel 5.10 以上版本，支持自动拓扑发现与带宽分配。  
- **NVMe 直连工具**：提供 `nvme-cli` 兼容层，支持 SSD 健康检查、固件升级。  
- **BMC 管理**：通过 `ipmitool` 或 Redfish 客户端即可实现远程监控、日志下载。

## 部署建议

1. **硬件检查**：确认 PCIe Switch 与 GPU 卡的固件版本匹配，确保 Switch 支持所选 PCIe 规格（4.0/5.0）。  
2. **温度预留**：在机箱内部预留足够的风道，避免风扇在高负载时出现风压不足导致温度异常。  
3. **热插拔测试**：在正式上线前进行 10 次以上的硬盘热插拔循环，验证灯态、SPGIO 与 BMC 报警的完整性。  
4. **固件升级**：使用昆仑固件提供的 OTA（Over‑The‑Air）功能，保持系统固件在最新安全补丁状态。  
5. **监控策略**：在运维平台上配置阈值报警（温度 > 80 °C、硬盘 I/O 错误率 > 0.1%）并关联自动化脚本，实现故障自愈。
