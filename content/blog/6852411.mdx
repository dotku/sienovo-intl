## 3. 进阶应用：文本级加密与隐写

因为 sng 能把图像完整映射为可编辑的文本，**文本本身就可以作为加密载体**。下面描述一种基于 sng 的简易隐写方案，适用于需要在公开渠道隐藏敏感图片的场景。

### 3.1 加密流程

1. **原始图像 → sng**  
   ```bash
   $ sng secret.png > secret.sng
   ```
2. **对 sng 文本进行混淆**（例如使用 `tr` 将字符随机置换，或使用自定义脚本进行字符级加密）。这里以 `openssl enc` 为例：
   ```bash
   $ openssl enc -aes-256-cbc -salt -in secret.sng -out secret.enc -k mypassword
   ```
3. **将加密后的文本再转为 PNG**（先解密，再交给 sng）：
   ```bash
   $ openssl enc -d -aes-256-cbc -in secret.enc -out secret2.sng -k mypassword
   $ sng secret2.sng > noise.png
   ```

生成的 `noise.png` 看起来像是一张噪声图，肉眼难以辨认其中隐藏的内容。只有掌握正确的密钥并按照逆向步骤解密，才能恢复出原始图像。

### 3.2 实际案例：毕业设计中的隐写实验

在作者的毕业设计中，利用上述两步转换（PNG → sng → 加密 → sng → PNG），成功将一幅彩色照片隐藏为噪声图，并通过自定义的密钥文件进行解密。该实验验证了 **“文本化图像 + 传统加密”** 的组合能够在不增加额外硬件负担的前提下，实现安全的图像隐藏。

## 4. 常见问题与调试技巧

| 问题 | 可能原因 | 解决办法 |
|------|----------|----------|
| `sng: command not found` | 未安装或未加入 PATH | 确认已通过源码编译或包管理器安装，或在 `~/.bashrc` 中添加 `export PATH=$PATH:/usr/local/bin` |
| 转换后 PNG 大小异常 | sng 文本格式错误（缺少换行、颜色码不完整） | 使用 `cat -A` 检查文本是否包含非法字符，确保每行仅包含 6 位十六进制颜色码 |
| 颜色替换未生效 | `sed` 正则未匹配到目标颜色 | 确认原始 PNG 中的颜色是否正好为 `#0000ff`（可能存在透明通道或颜色抖动），可先用 `grep -o '[0-9a-fA-F]\{6\}'` 检查文本内容 |
| 加密后生成的 PNG 只能显示噪声 | 解密步骤未正确执行 | 再次检查解密命令的密钥、盐值是否一致，确保解密输出的文件是合法的 sng 文本 |
