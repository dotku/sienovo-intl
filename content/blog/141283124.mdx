## 引言  

本文将详细介绍 **STOBER** 公司在德国普福尔茨海姆利用 **ARM+Codesys** 平台实现的全自动机器人装配案例。读者可以了解该案例中使用的硬件（RK3568/A40i/STM32、SCARA 机器人、旋转分度台等）以及 **CODESYS** 作为软 PLC 在运动控制、EtherCAT 同步以及人机界面中的核心作用，帮助您在智能生产线或高端装备研发中快速落地类似方案。

## 案例概览  

在欧洲尤其是德国，借助于全自动机器人进行工件组装早已不是什么新鲜事儿，比如普福尔茨海姆市（德国城市）的 **STOBER**（斯德博）公司制造的 **无编码器 LeanMotor** 在世界市场上位居第一。他使用全自动机器人元件安装系统，为这种高效驱动解决方案生产转子。该系统完全自动化，是基于内部开发的最新技术以及德国 **CODESYS** 的控制方案作为核心支持。再加上其广泛的产品组合，STOBER 能够实现各种各样的应用程序。  

![图片](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/141283124/01_2874fa1a4b00d69fc5f13b75e9d64f5c.jpeg)

## LeanMotor 的技术优势  

- **无编码器驱动器** 能提供始终如一的输出，比非同步驱动器更轻，电机比伺服电机更小，更具成本效益和坚固性。  
- 通过 **全扭矩控制**，速度和扭矩可以任意控制，效率高达 **96%**，LeanMotor 符合 **IE5** 效率等级。  
- 与 **SC6**、**SI6** 控制器结合后，**速度偏差** 小于 **1%**，**定位精度** 为 **±1 度**。  

这些指标在实际装配线上表现出极高的重复定位能力，为后续机器人路径规划提供了可靠的基础。

## 自动装配系统的硬件构成  

为了生产新型 LeanMotor 的圆形坯料，在 **CODESYS** 集团的协助下 STOBER 公司开发了一种全自动安装系统，包括 **三个 SCARA 机器人** 和一个带有 **五个工位的旋转分度台**，用于各个加工步骤。  

![图片](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/141283124/02_b3da6981b95764c6619f848a0f88047b.jpeg)

- **低背隙行星齿轮或螺旋锥齿轮减速机**：用于需要高精度和重复精度的任何地方，尤其是机器人和旋转分度台中的轴。  
- **STOBER MC6 运动控制器**：负责集中控制，特别适配 **EtherCAT** 的同步控制，并在此过程中处理整个人机界面（HMI）。  

## CODESYS 在系统中的角色  

### 嵌入式 Runtime  

**STOBER MC6 运动控制器** 基于 **CODESYS** 的嵌入式 Runtime 开发，**CODESYS** 为应用程序员保持了尽可能简单的界面。程序员可以：

1. 使用 **点对点 (PtP)** 直接定位机器人。  
2. 在必要时使用程序序列中的 **示教位置**。  
3. 通过 **CODESYS** 提供的库功能块完成 **三个 SCARA 机器人的路径规划和插补** 等所有计算。  

### 软 PLC 的跨平台特性  

**CODESYS** 本质上是一款软 PLC，除了可以被移植到主流的嵌入式开发板（**ARM、×86、PowerPC、TriCore** 等），支持 **Linux、VxWorks、QNX** 嵌入系统之外，还可以直接安装在 PC 端，直接把您的电脑变成一台显控一体的控制器。  

- **可视化**：提供直观的图形化界面，便于调试和监控。  
- **运动控制系统**：内置丰富的运动控制库，支持多轴同步、插补算法等。  
- **总线协议**：支持 **30 多种** 主流现场总线（如 EtherCAT、PROFINET、CANopen 等），满足工业现场的多样化需求。  
- **OPC / OPC UA**：基于标准 OPC 协议和 OPC UA 协议，配合 **PLCHandler** 第三方客户端，实现与 SCADA、MES 等上位系统的无缝集成。  

### OEM 与生态联盟  

**CODESYS** 的众多 OEM 合作伙伴形成了 **CODESYS 自动化联盟**，能够为其提供硬件设备。使用这样一种自动化开发平台开发的产品必然具备 **稳定、可靠、高效** 的特性，在提升产品品质的同时必然会降低开发成本。

## 实际部署要点  

| 步骤 | 关键技术点 | 注意事项 |
|------|------------|----------|
| 1. 硬件选型 | RK3568/A40i/STM32 等 ARM 开发板，SCARA 机器人，旋转分度台 | 确认板卡的 **EtherCAT** 主从兼容性 |
| 2. 系统集成 | 将 MC6 控制器通过 EtherCAT 接入 CODESYS Runtime | 采用 **同步时钟**（Distributed Clocks）确保多轴同步 |
| 3. 运动规划 | 使用 CODESYS 库功能块进行 PtP、示教、路径插补 | 通过 **仿真模式** 先验证轨迹，避免实际碰撞 |
| 4. HMI 开发 | 在 CODESYS 中配置可视化页面，展示机器人状态、错误报警 | 采用 **分层设计**，将关键参数放在主界面，次要信息放在子页面 |
| 5. 调试与验证 | 通过 CODESYS 在线监控功能查看实时轴位置、速度、扭矩 | 使用 **日志记录**（Trace）捕获异常，便于后期分析 |
| 6. 交付运行 | 进行 **长时间运行测试**（48h+），验证可靠性 | 检查 **温度、功耗** 是否在额定范围内 |
