#### 1.3 虚拟机键盘切换技巧

在 Windows 虚拟机（如 VirtualBox、VMware）中，`Ctrl+Alt+F1~F7` 常被宿主系统拦截，导致无法切换虚拟终端。社区中流传的组合键 `Ctrl+Alt+Space+F1`（进入命令行）和 `Ctrl+Alt+Space+F7`（返回图形）在多数情况下有效，原因是 `Space` 键在虚拟机软件中被用作“释放键”，可以让宿主系统把后续的功能键交给客机。

> **提示**：如果上述组合仍然失效，尝试在虚拟机设置里打开“键盘捕获”或使用“全屏模式”，这样 `Ctrl+Alt+F1` 系列键就能直接传递给 Linux。

### 2. 从命令行模式切换回图形界面

**原始回答**  
> **答**：`startx`、`xdm`、`gdm`、`kdm` 都行，如果进不了 X WINDOWS 可能是 X WINDOWS 设置不正确，这时可耻下场用 `Xconfigurator` 命令对其进行重新设置!!  
> - `logout`  注销  
> - `reboot`  重启  
> - `startx`  启动图形界面  
> - `gdm`  进入 GNOME 图形界面  
> - `kdm` 进入 KDE 登陆界面  

#### 2.1 常见启动方式的区别

| 命令 | 说明 | 适用场景 |
|------|------|----------|
| `startx` | 手动启动 X Server 并运行默认的 `.xinitrc` 脚本 | 轻量桌面、没有完整的显示管理器 |
| `xdm` | X Display Manager，最早的显示管理器 | 老旧系统或需要最小化依赖的环境 |
| `gdm` | GNOME Display Manager，提供 GNOME 桌面登录 | GNOME 桌面环境 |
| `kdm` | KDE Display Manager，提供 KDE 桌面登录 | KDE 桌面环境（已被 `sddm` 取代） |
| `sddm` | Simple Desktop Display Manager，现代 KDE/Plasma 默认 | KDE Plasma 5+ |
