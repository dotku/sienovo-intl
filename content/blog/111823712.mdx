该错误表明 `unrar` 包在当前的源中没有可用的安装候选，导致无法直接通过 `apt` 完成安装。下面提供一种基于官方 RAR Lab 下载的离线安装方案，帮助在 Ubuntu 16.04（以及其他兼容的 Linux 发行版）上顺利部署 RAR/UnRAR。

## 为什么会出现 “no installation candidate”

- **仓库不完整**：Ubuntu 16.04 默认的 `main`、`universe`、`multiverse` 仓库中并未收录官方的 `unrar`（非自由版），而是提供了 `unrar-free`（功能受限的开源实现）。
- **版权限制**：原版 `unrar` 属于专有软件，Ubuntu 出于版权合规的考虑不直接提供。
- **源同步问题**：如果系统的 `sources.list` 中缺少 `multiverse` 源，`apt` 也会找不到对应的包。

如果只想使用开源的 `unrar-free`，可以直接执行 `sudo apt-get install unrar-free`；但如果需要完整的 RAR 功能（如创建 RAR 压缩包），仍需手动安装官方提供的二进制。
