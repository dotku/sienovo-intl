## 引言  
在工业自动化和科研实验中，**基于PC的测量和控制系统**已经成为实现高精度、可定制化数据采集的主流方案。本文将围绕NI（National Instruments）提供的硬件与软件生态，系统阐述该类系统的基本架构、核心特性以及在实际工程中的应用方式，帮助读者快速上手并进一步发挥其在自动化、信号调理和高级分析方面的潜力。

## 1. 系统整体架构  

### 1.1 硬件连接方式  
在基于PC的测量和控制系统中，NI硬件产品通过 **USB** 或 **以太网** 连接到PC或笔记本电脑。两种常见的架构如下：

| 架构 | 描述 |
|------|------|
| **多功能I/O设备直接连接** | 将NI的多功能I/O卡（如USB‑6363）直接插入PC的USB端口，适用于通道数不多、对空间要求较低的场景。 |
| **CompactDAQ机箱+调理模块** | 将CompactDAQ机箱通过USB或以太网接入PC，在机箱内部插入经过信号调理的I/O模块，能够直接连接各种传感器，实现更高的可定制性和模块化管理。 |

> ![](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/135447184/01_20e4b03707f759134aa87e89eee06e3e.png)

### 1.2 CompactDAQ的优势  
CompactDAQ提供了 **模块化的信号调理架构**，无论是温度、压力、应变还是高速模拟信号，都可以通过对应的调理模块直接与传感器对接。部分CompactDAQ机箱还支持 **时间敏感网络（TSN）**，TSN是以太网标准的更新版本，可在多个机箱之间提供同步和数据传输，从而省去单独的定时和触发线。

## 2. 软件生态  

### 2.1 DAQExpress™ 快速上手  
所有基于PC的DAQ硬件均附带 **DAQExpress™** 配套软件。该软件的核心价值在于：

- **快速配置**：用户可以在几分钟内完成多通道测量的设置，无需编写代码。  
- **即时浏览**：测量数据实时显示，支持波形放大、光标测量等基本分析功能。  
- **数据保存**：支持多种文件格式（TDMS、CSV）导出，便于后续处理。  

借助DAQExpress，工程师能够 **快速捕获和探索数据**，并在此基础上使用 **LabVIEW** 实现测量自动化、执行高级分析和集成第三方仪器。

### 2.2 NI-DAQmx 驱动的高级功能  
NI-DAQmx 驱动软件功能远强于基本的数据采集驱动，为数据采集和控制应用开发提供了更高的效率和性能。其主要特性包括：

- **统一 API**：通过单一的函数调用即可完成从设备发现、通道配置、时序设置到底层操作系统交互的全部工作。  
- **跨平台兼容**：同一套代码可以在不同型号的NI硬件上运行，无需修改。  
- **多语言支持**：NI-DAQmx 可与 **LabVIEW、DAQExpress、LabWindows™/CVI、C/C++、Visual Basic、Visual Basic .NET** 和 **C#** 一起使用，满足不同开发者的语言偏好。  

### 2.3 精准定时与 STC3 计时芯片  
NI测量产品和专利技术（例如 **STC3计时芯片**）为模拟设计提供了出色的质量和精度，使您可完全信赖测量数据。STC3 通过硬件级别的时钟同步，确保在高速采样或多通道同步采集时，时序误差保持在亚微秒级别。

## 3. 实际应用场景  

### 3.1 实验室数据采集  
在实验室中，研究人员常需要对温度、压力、流量等多种物理量进行同步采集。使用CompactDAQ机箱+调理模块的方式，可以直接将热电偶、压阻式传感器等接入系统，利用NI-DAQmx 在LabVIEW 中编写简短的循环程序，实现 **实时监控 + 自动记录**。

### 3.2 生产线质量检测  
生产线上对产品尺寸或表面质量的快速检测往往要求 **低延迟** 与 **高可靠性**。通过 **TSN** 同步多台CompactDAQ机箱，能够在不同检测站点之间保持毫秒级同步，避免因网络延迟导致的误判。配合DAQExpress 的 **即时浏览** 功能，操作员可以在现场快速判断合格率。

### 3.3 嵌入式边缘AI  
在边缘AI场景下，常需要将采集到的传感器数据送入机器学习模型进行实时推断。NI-DAQmx 的 **C/C++** 接口可以与 **TensorRT**、**OpenVINO** 等推理引擎无缝对接，实现 **采集 → 预处理 → 推断** 的闭环。由于驱动层已经完成了高效的缓冲管理，整体系统的 **吞吐率** 与 **延迟** 能够满足工业级实时要求。

## 4. 部署与调试要点  

| 步骤 | 关键点 |
|------|--------|
| **硬件连接** | 确认USB/以太网线缆质量，使用NI MAX检测设备是否被系统识别。 |
| **驱动安装** | 安装最新的 **NI-DAQmx** 与 **NI Measurement & Automation Explorer (MAX)**，确保驱动版本与硬件固件匹配。 |
| **软件配置** | 在DAQExpress 中创建 **Task**，选择对应的通道类型（模拟输入、数字输出等），并设置采样率与缓冲大小。 |
| **同步设置** | 若使用TSN，需在NI MAX 中启用 **Time Sensitive Networking**，并在网络交换机上配置对应的 **IEEE 802.1AS** 同步协议。 |
| **代码验证** | 使用LabVIEW 示例代码或C示例程序进行 **单通道** 采集验证，确认波形、时序与预期一致后再扩展到多通道。 |
| **数据存储** | 推荐使用 **TDMS** 格式保存原始数据，便于后期在LabVIEW 中进行批量分析或在Python 中使用 **nptdms** 库读取。 |

## 5. 小结  

基于PC的测量和控制系统通过 **NI硬件 + DAQExpress + NI-DAQmx** 的完整生态，为工程师提供了 **快速、精准且经济实惠** 的数据采集方案。无论是实验室原型验证、生产线质量监控，还是边缘AI 推断，均可依托其模块化的 **CompactDAQ** 架构、可靠的 **STC3计时芯片**、以及支持 **TSN** 的同步能力，构建出符合特定需求的测量系统。
