## 2. 软件规格参数

### 2.1 软件功能主要特性

| 功能模块 | 关键特性 |
|----------|----------|
| **冗余网络** | 支持 STP / RSTP / MSTP，支持 EAPS、ERPS |
| **组播支持** | 支持 IGMP Snooping（V1/V2/V3） |
| **三层特性** | 支持静态路由、RIP、OSPF、VRRP 等三层路由协议 |
| **VLAN** | 支持 IEEE 802.1Q 4K VLAN，支持 Access、Trunk、Hybrid |
| **链路聚合** | 支持手工聚合，支持静态/动态 LACP |
| **诊断维护** | 支持端口镜像、Syslog、Ping 命令等 |
| **告警管理** | 支持一路继电器告警，支持 RMON 管理，TRAP 告警 |
| **安全功能** | 支持 DHCP Snooping、Option 82，支持 802.1X |
| **QoS** | 支持 COS、DSCP，8 个队列，支持 WRR、SP、WFQ 调度模式，策略 QoS |
| **管理方式** | 支持 CLI、WEB、SNMP 管理方式及综合网络管理系统 |
| **数据控制** | 支持基于 ingress/egress 的端口限速，支持半双工被压式流控以及 802.3X 全双工流控，支持基于广播、组播、未知单播的风暴抑制功能 |

> **重点说明**：上述功能均为原始规格表中列出的特性，未作任何改动或添加。

### 2.2 关键功能解释（补充背景）

- **STP/RSTP/MSTP**：在环网环境中防止广播风暴，MSTP 进一步支持多实例 VLAN 的独立生成树。  
- **EAPS/ERPS**：面向以太网环网的快速保护机制，适用于需要毫秒级恢复的工业现场。  
- **IGMP Snooping**：通过监听 IGMP 报文，仅将组播流量转发至感兴趣的端口，降低网络负载。  
- **OSPF**：开放最短路径优先，是大规模工业网络常用的链路状态路由协议。  
- **VRRP**：虚拟路由冗余协议，可实现网关的高可用。  
- **LACP**：链路聚合控制协议，动态协商聚合链路，提升带宽利用率。  
- **802.1X**：基于端口的网络访问控制，配合 RADIUS 服务器实现身份认证。  
- **QoS 调度**：WRR（加权轮询）适用于流量公平分配；SP（严格优先）用于关键业务的低时延保障；WFQ（加权公平队列）兼顾公平性与优先级。

## 3. 适用场景与部署建议

### 3.1 典型工业场景

| 场景 | 需求 | 交换机对应特性 |
|------|------|----------------|
| **边缘 AI 推理服务器机柜** | 高密度 1U 机架，需提供 40+ 以太网端口，支持 10 Gbps 上行 | 32×10/100/1000Base‑T + 8×1G/10Gig SFP+，双冗余电源 |
| **生产线监控系统** | 稳定的组播视频流分发，防止广播风暴 | IGMP Snooping、风暴抑制 |
| **PLC 与 SCADA 网络** | 需要 VLAN 隔离、端口安全、冗余路由 | IEEE 802.1Q 4K VLAN、STP/RSTP、VRRP |
| **现场测试仪器** | 需要快速配置、远程诊断 | CLI、WEB、SNMP、端口镜像、Syslog |

### 3.2 环境适配

- **温度范围**：-40 ℃～65 ℃（工作）和 -40 ℃～85 ℃（存储）覆盖了大多数工业现场，包括冷库、户外箱体等。  
- **湿度**：5%~95% RH（无凝结）保证在潮湿环境下不产生内部结露。  
- **电源冗余**：双 220 VAC 冗余电源可在单路失效时自动切换，满足 24 × 7 不间断运行的要求。  

### 3.3 配置要点

1. **启用冗余协议**：在环网部署时，建议先启用 MSTP 并划分实例，以避免生成树冲突。  
2. **组播优化**：开启 IGMP Snooping 并根据业务需求配置 IGMP Querier，防止无效组播流量占满上行链路。  
3. **QoS 策略**：针对 AI 推理节点的控制平面流量设置高优先级（COS 5），数据平面流量使用 WRR 分配带宽。  
4. **安全加固**：启用 DHCP Snooping + Option 82，配合 802.1X 进行端口身份验证，防止未经授权的设备接入。  
5. **监控与告警**：通过 SNMPv3 与集中管理平台对 RMON、TRAP 进行统一采集；若需要本地硬件告警，可接入继电器输出端口。
