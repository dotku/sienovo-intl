## 引言

随着城市高层建筑的快速增长，电梯的安全与可靠性成为公共安全的重要组成部分。传统的电梯维保模式往往依赖定期巡检，难以及时发现潜在故障，导致故障率偏高、维修成本居高不下。基于物联网（IoT）技术的 **电梯按需维保** 方案通过实时采集、分析电梯运行数据，实现对安全隐患的预测式排查，从而提升维保的科学性、针对性和精准性。本文将围绕 **物联网网关** 在该方案中的关键作用展开，详细解析其功能特点、技术优势以及实际部署要点，帮助读者全面了解电梯物联网网关在提升电梯运行安全方面的价值。

## 物联网网关在电梯按需维保中的定位

物联网网关是连接电梯内部传感器与云平台的桥梁。它通常安装在电梯轿门或机房内部，负责：

- **实时采集** 各类高精度传感器数据（如门磁、电流、人体感应等）；
- **本地预处理** 与短期存储，确保在网络中断时仍能保留关键信息；
- **安全可靠的通信**（4G/NB‑IoT 等）将数据上报至监控平台和云端；
- **远程指令下发**，实现对电梯状态的远程监控与控制。

通过上述功能，网关实现了从“被动巡检”向“主动预警”的根本转变，为维保人员提供了精准的维修依据。

## 功能特点详解

### 1. 完整的运行状态监测

> **电梯上下运行信号、停梯信号、平层信号、轿门开关信号、电梯所处楼层、电梯速度、电梯单次运行距离；**

这些信号覆盖了电梯的基本运动过程。网关通过内置的 **门磁传感器**、**位置编码器** 与 **速度计**，实时捕获电梯的上下行、停梯、平层等状态，并将楼层、速度、运行距离等关键指标以结构化数据形式上报。维保系统可以基于这些数据绘制电梯使用曲线，识别高频使用的楼层或异常停机模式，为后续的负荷评估提供依据。

### 2. 实时故障监测

> **实时监测电梯故障信息（困人、卡层、梯门异常、冲顶蹲底）；**

网关集成了 **人体感应器** 与 **电流传感器**，能够在电梯出现异常电流或门异常开启时立即触发告警。例如，当电梯在运行过程中出现 **困人**（轿厢内有人但门未关闭）或 **卡层**（电梯在某层停留异常时间）时，网关会将故障事件实时推送至监控平台，维保人员可在手机或电脑端第一时间获知并安排现场处理，显著降低事故风险。

### 3. 数据存储与维保信息记录

> **存储电梯的各类运行数据、故障数据、电梯维保工作信息等；**

为保证数据的完整性，网关内部配备了 **工业级 NAND Flash**，能够缓存最近 **100 条**（或更多）运行与故障记录。即使在 4G 网络暂时不可用的情况下，数据仍然保存在本地，待网络恢复后统一上报。这种本地缓存机制确保了数据不因通信中断而丢失，为后续的故障追溯与分析提供了可靠依据。

### 4. 可靠的远程传输

> **通过 4G 无线网络将电梯运行实时状态数据传输至电梯运行实时监控平台并且可以同时上报至云平台。**

4G 网络的覆盖范围广、带宽充足，能够满足电梯实时数据（包括视频、传感器流）的上传需求。网关在采集后会对数据进行 **压缩与加密**（符合行业数据安全规范），随后通过 **HTTPS** 或 **MQTT** 协议推送至监控平台和云端。平台侧可实现多租户数据隔离、历史数据查询、异常报警推送等功能，为维保公司提供统一的运维视图。

## 物联网网关 IoT‑G40A 技术优势

> **电梯物联网网关 IoT‑XM 作为业界首选，在技术方面优势显著，该款物联网网关已通过了多项行业检测，满足包括工作电源、隔离措施、通信端口及传输协议、数据传输误差、备用电源、电源波动适应能力（交流标称值±7%，直流标称值±5%）、电源输入保护（可靠接地、过流保护、漏电保护）、监测终端组成及整备质量、时钟溯源性、数据上报时间（不大于 5S）、设备运行状态监测（不大于 1s）、存储与统计（至少保存最近 100 条信息）、数据加密与管理权限、监测终端组件质量、电气配线、外壳防护等级、绝缘电阻（正常大气条件下不小于 100MΩ）、介电强度、阻燃性能、噪声干扰、环境适应性（高低温、冲击、振动、跌落等）、电磁兼容性等在内数十项技术要求。**

### 1. 稳定的电源与供电适应性

- **交流电源波动 ±7%**、**直流电源波动 ±5%** 的容忍度，使网关能够在电梯机房常见的电压波动环境中保持正常工作。
- **备用电源**（如内置锂电池）保证在主电源失效时仍能继续采集并缓存关键数据，防止信息丢失。

### 2. 完备的安全防护

- **可靠接地、过流保护、漏电保护** 确保设备在电气故障时不会对电梯系统或人员安全造成二次危害。
- **绝缘电阻 ≥ 100MΩ** 与 **介电强度** 检测，满足 IEC 60601‑1 等医疗/安全标准的绝缘要求。

### 3. 高速、低延迟的数据上报

- **数据上报时间 ≤ 5 秒**、**运行状态监测 ≤ 1 秒**，保证监控平台能够实时获取电梯状态，及时触发告警。

### 4. 强大的环境适应能力

- **防护等级**（IP54/IP65 等）以及 **阻燃、抗冲击、抗振动、抗跌落** 的机械设计，使网关能够在电梯井道、机房等恶劣环境中长期可靠运行。

### 5. 数据安全与权限管理

- 支持 **TLS/SSL 加密**、**基于角色的访问控制（RBAC）**，确保数据在传输与存储过程中的机密性与完整性，防止未经授权的访问。

> 以上技术优势为电梯按需维保提供了坚实的硬件基础，使得整个系统在可靠性、实时性与安全性上均达到行业领先水平。

## 部署与配置要点

### 1. 安装位置与布线

网关一般固定在 **电梯轿门** 或 **机房侧壁**，确保传感器能够覆盖门磁、楼层编码器等关键部位。布线时需遵循 **电气配线规范**，使用 **屏蔽双绞线** 以降低电磁干扰；同时，所有接线点应做好 **防潮、防腐** 处理，避免长期运行中的氧化问题。

### 2. 传感器校准

- **门磁传感器**：需在安装后进行闭合/打开状态的校准，确保误报率低于 0.5%。
- **电流传感器**：通过现场负载测试校准量程，以获得准确的功耗数据。
- **人体感应器**：在不同光照、温度条件下进行灵敏度调节，避免误触发。

### 3. 网络接入配置

网关支持 **4G LTE**、**NB‑IoT** 双模通信。配置时应：

1. 在 **SIM 卡** 插槽中插入已开通数据流量的运营商卡；
2. 通过网关的 **Web UI**（或移动 App）进入 **网络设置**，填写 **APN**、**用户名/密码**（如运营商要求）；
3. 启用 **自动网络切换** 功能，以在 4G 信号弱时自动切换至 NB‑IoT，保证数据上报的连续性。

> 配置示例（仅示意）  
> ```
> 网络模式: 自动切换
> APN: internet
> 用户名: user123
> 密码: pass123
> ```

### 4. 数据上报与平台对接

- **上报协议**：推荐使用 **MQTT**（TLS 加密）或 **HTTPS**（POST JSON），平台方需提供对应的 **Topic** 或 **API Endpoint**。
- **数据格式**：统一采用 **ISO 8601** 时间戳、**JSON** 键值对，如：
  ```json
  {
    "elevator_id": "E12345",
    "timestamp": "2026-05-27T14:30:12Z",
    "floor": 12,
    "door_status": "closed",
    "speed": 1.2,
    "fault_code": 0
  }
  ```
- **离线缓存**：当网络不可用时，网关会将数据写入本地 Flash，待网络恢复后自动批量上报，确保 **不丢失**。

### 5. 远程运维与固件升级

网关具备 **OTA（Over‑The‑Air）** 升级能力，维保中心可通过云平台推送最新固件。升级前建议：

- 确认 **电梯处于停梯状态**，避免因升级导致的通信中断影响安全监控；
- 预留 **至少 10%** 的 Flash 空间用于固件备份，以防升级失败时回滚。

## 小结

物联网网关是电梯按需维保系统的核心枢纽，它通过 **高精度传感**、**可靠的本地存储**、**低延迟的远程传输** 与 **严格的安全防护**，实现了对电梯运行状态的全方位实时监控。凭借 **IoT‑G40A** 的多项技术优势，网关能够在复杂的电梯环境中保持长期稳定运行，为维保人员提供精准的故障预警和科学的维修依据。未来，随着 AI 分析模型的进一步嵌入，网关将不仅仅是数据采集终端，更会成为电梯安全智能决策的关键节点。
