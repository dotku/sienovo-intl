1.  Goal: Implement a real-time kernel using TI Linux 4.4.19 (which runs normally on the board) + Xenomai, i.e., port Xenomai to Linux 4.4.19.

Target board: Xinmai TI AM5728 Development Board

![](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/113574231/01_ce3a8caff441b9260386b01b8b7f28cb.png)

1.  Problem: After applying the ipipe patch, a compilation error occurred in `timer.c`. After modification, it compiled successfully.
2.  Reference tutorial: [https://www.cnblogs.com/wsg1100/p/12864199.html](https://www.cnblogs.com/wsg1100/p/12864199.html)

4. Compile user-space libraries:

[xmtech@xmtech-server01:~/xenomai/xenomai-3.1](mailto:xmtech@xmtech-server01:~/xenomai/xenomai-3.1)

./configure CFLAGS="-march=armv7-a -mtune=cortex-a15 -mfloat-abi=hard -mfpu=neon -ffast-math" --host=arm-linux-gnueabihf --with-core= cobalt \--enable-smp --enable-pshared

[xmtech@xmtech-server01:~/xenomai/xenomai-3.1](mailto:xmtech@xmtech-server01:~/xenomai/xenomai-3.1)  mkdir build_fs

make DESTDIR=~/xenomai/xenomai-3.1/build_fs install

mount /dev/sdb2 /opt/tmp

[xmtech@xmtech-server01:~/xenomai/xenomai-3.1](mailto:xmtech@xmtech-server01:~/xenomai/xenomai-3.1) cp -r ./build_fs /opt/tmp/usr
