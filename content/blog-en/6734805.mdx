Open /etc/vimrc with vim  
vi /etc/vimrc  
Add the following lines at the end:  
set number  
set tabstop=4  
If  
set ai  
already exists in the vimrc file but is commented out with double quotes, simply remove the double quotes.  
If set ai does not exist, add it at the end.

I'm using CentOS 5.3. After applying the above settings, I found that Vim for regular users was properly configured. However, when the root user used the vi command, it launched vi instead of vim, so auto-indentation and line numbers were not enabled. Here's how to fix it:  
Open the root user's .bashrc file  
vi /root/.bashrc  
Add the following line:  
alias vi=vim  
Save the file, log out of the root account, then log back in, and the issue will be resolved.
