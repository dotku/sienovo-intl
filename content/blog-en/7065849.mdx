Memory occupied by programs compiled with C/C++:
1. Stack — Automatically allocated and deallocated by the compiler, used to store function parameters, local variable values, etc. Its operation is similar to the stack data structure.
2. Heap — Generally allocated and deallocated by the programmer. If not deallocated by the programmer, it might be reclaimed by the OS when the program ends. Note that it's different from the heap data structure; its allocation method is more similar to a linked list, haha.

Stack: In Windows, the stack is a data structure that expands towards lower memory addresses and is a contiguous memory region. This means that the stack's top address and maximum capacity are predetermined by the system. In Windows, the stack size is 2MB (some say 1MB, but it's a compile-time constant). If the requested space exceeds the remaining stack space, an overflow will be prompted. Therefore, the space obtainable from the stack is relatively small.
Heap: The heap is a data structure that expands towards higher memory addresses and is a non-contiguous memory region. This is because the system uses a linked list to store free memory addresses, which are naturally non-contiguous, and the linked list traversal direction is from lower to higher addresses. The heap size is limited by the effective virtual memory in the computer system. Thus, the space obtainable from the heap is more flexible and larger.
