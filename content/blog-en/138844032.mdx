PCI bus Mil-1394B full-featured card, supporting 1 node (CC) with all CC, RN, and BM functionalities.

      The 1394 full-featured card is used to simulate CC, RN, and BM node functionalities within a 1394 bus. It supports features such as bus topology viewing, data logging, and error injection, enabling the construction of a 1394 bus network and data transmission across it. The core functionality of the 1394 three-node simulation card is implemented based on an FPGA, which handles the parsing of the 1394B bus standard AS5643 protocol. This design offers fast FPGA logic function startup, strong protocol processing capability, small size, and light weight, meeting the product's testing requirements in laboratory environments.

![](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/138844032/01_3fcd89aea20e7a3f6589b5ea2b12610b.png)

·   Supports 1394 network communication, using the standard SAE AS5643 protocol format;

·   Employs transformer coupling to provide Beta mode 1394B communication ports;

·   Supports S100B, S200B, and S400B transmission modes, with configurable modes;

·   Provides transmit and receive functionalities for CC/RN nodes, and BM monitoring functionality;

·   Received messages are time-stamped; the timestamp is 64-bit with 1us precision.

·   Supports bus data monitoring, bus topology display, and data parsing;

·   Supports sending 32 event response messages per cycle;

·   Supports reporting received data attributes (channel number, message ID, message offset, message data CRC error status, message VPC error status, message length error status, message rate);

·   Supports asynchronous stream message error injection (header CRC, data CRC, data VPC);

·   Supports CC node STOF error injection (header CRC, data CRC, data VPC, error cycle);

·   Supports initiating bus reset and reset storm configuration;

·   Supports node port disable and enable operations.
