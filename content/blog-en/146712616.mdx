1.  Introduction

Built upon the Centec CTC7132 SOC solution, this module exposes 32 channels of 10G SerDes interfaces, 1 PCIe x1 interface, 2 SGMII management interfaces, 3 UART interfaces (1 debug serial port + 2 functional serial ports), 4 I2C interfaces (2 for SOC, 2 for PPU), 5 SMI interfaces (1 management port + 4 service ports), 1 100MHz PCIe differential reference clock output interface, a serial LED signal group, etc., via board-to-board high-speed connectors. When used with a carrier board, it can flexibly provide multi-rate, multi-form interfaces to meet diverse application requirements.

2.  Hardware Parameters

1.  Core board onboard DDR4 capacity: 2GB;
2.  Core board onboard eMMC capacity: 8GB;
3.  Provides 32 channels of 10G SerDes signal interfaces, supporting SGMII/QSGMII/XAUI/XFI/10G-KR and other modes;
4.  Provides 2 SGMII management interfaces;
5.  Provides 1 PCIe interface;
6.  Provides system operation indicator signals, power indicator signals, over-temperature alarm signals, and configurable service port serial LED stream signals;
7.  Provides temperature monitoring function, supports over-temperature power-off, and configurable temperature thresholds;
8.  For detailed signal definitions, please refer to Appendix A.

3.  Hardware Implementation Scheme

The core board hardware solution mainly consists of the CTC7132 SOC system, power supply, clock, and other system components. It exposes 32 channels of 10G SerDes interfaces, 2 SGMII management interfaces, 1 PCIe 2.0 x1 interface, 3 UART interfaces, 4 I2C interfaces, 5 SMI interfaces, 1 100MHz PCIe differential reference clock output interface, 1 serial LED signal group, and GPIO signals, etc., via board-to-board high-speed connectors. 4.  Software Features

1.  Supports 802.1Q VLAN;
2.  Supports VLAN based on IP address and MAC address;
3.  Supports STP/RSTP/MSTP protocols;
4.  Supports ERPS protocol;
5.  Supports static link aggregation, supports LACP aggregation protocol;
6.  Supports LLDP link layer discovery protocol;
7.  Supports IGMP Snooping function;
8.  Supports static routing, supports RIP routing protocol, supports OSPF routing protocol;
9.  Supports DHCP Server, DHCP Client;
10. Supports PIM protocol;
11. Supports ACL (filtering, policing, port mirroring), provides filtering based on 5-tuple (source MAC address, destination MAC address, source IP address, destination IP address, port);
12. Supports 802.1X authentication (basic port) function;
13. Supports QoS congestion management, capable of data classification and prioritization;
14. Supports port flow control, supports port isolation;
15. Supports configuration and maintenance via serial port, Telnet, SSH;
16. Supports SNMP V2/V3 protocols;
17. Supports WEB page operation configuration;

12. Software Adaptation

The core board supports users adapting software themselves or the designer providing targeted software adaptation according to the user's carrier board design requirements. For specific matters regarding targeted software adaptation, please consult the core board designer's sales personnel. Additionally, targeted software adaptation must adhere to the following constraints in terms of carrier board design:

1.  PHY devices support CTC21101, CTC21104, CTC21108, YT8618, YT8614, YT8521 series products, etc.;
2.  When SerDes is routed to the carrier board for port mode configuration, it is necessary to confirm with the core board designer in advance whether there are any constraints;
3.  When SerDes is routed to the carrier board, if P/N inversion routing needs to be adjusted, it is recommended that RX and TX are inverted simultaneously to maintain synchronization;
4.  When the SMI bus manages PHY chips of different rates, the specific connection method needs to be confirmed with the core board designer;
5.  The carrier board design needs to include an EEPROM device (recommended model: FM24C64D-SO-U-G), with its address configured to 0x57, for storing actual adapted service port configuration information;
6.  The carrier board can optionally include an RTC device based on requirements. The recommended device model is Xingweifan SD2405ALPI-G;
7.  When placing RTC and EEPROM devices on the carrier board, they need to be connected to the CPU_I2C0 signal definition group. The core board's CPU_I2C0 already occupies addresses 0x48 (temperature sensor) and 0x56 (EEPROM). If more devices need to be placed, care must be taken to avoid address conflicts, and chain-like PCB routing is recommended;
