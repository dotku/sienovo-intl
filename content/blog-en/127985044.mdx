1 Summary
PLC controllers and remote I/O are increasingly communicating in current production environments. In some sites, controllers and remote I/O stations use a unified communication protocol, while in others, remote I/O stations may use different protocols, primarily due to product limitations from various manufacturers.
1.1 Solution
In a certain site in Henan, the controller uses a Siemens S7-1500 series PLC, and the modules for the remote I/O stations are all from Inovance AM600 medium-sized series PLCs. All I/O modules are connected to an AM600-RTU-ECAT coupler. The Inovance coupler module supports the EtherCAT protocol when operating normally, functioning as an EtherCAT slave. The monitoring system, a Siemens S7-1500 series PLC, supports the PROFINET protocol. The protocol inconsistency prevents direct communication for data exchange. A third-party protocol conversion module is required to enable data exchange between different protocols. In this case, our self-developed EtherCAT Master to PROFINET Slave gateway module ET300 is used to achieve comprehensive data interaction between the PLC and remote I/O.

### Features

*   Fast cyclic I/O data replication between the two networks (10-15 ms)
*   Verified and tested with all PLC manufacturers
*   Supports up to 512 bytes of input and output data in each direction
*   Ability to build web pages to display and control factory floor processes using data from other connected devices
*   Quick and dynamic transfer of fieldbus data to Microsoft Windows-based SCADA/HMI/enterprise-level systems via the included Anybus OPC server
*   Optional control and status information added to I/O data for diagnostic purposes
*   Robust standalone housing for harsh industrial environments
