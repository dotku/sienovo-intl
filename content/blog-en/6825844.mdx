To reduce the complexity of network design, most networks adopt a stack of overlapping layers. Each layer is built upon the one below it and serves the layer above, while hiding its internal implementation details from higher layers. The fundamental idea is that a specific piece of software (or hardware) provides a service to users while concealing the internal state and algorithmic details.

The n-th layer in one machine communicates with the corresponding n-th layer in another machine. The rules and conventions used in this communication are collectively known as the protocol of the n-th layer. In other words, a protocol is an agreement between communicating parties on how communication should be carried out. Violating such an agreement may still allow communication to proceed, but it becomes extremely difficult—much like interactions between people.

Interfaces between adjacent layers must be clearly defined to specify the primitive operations and services that the lower layer offers to the upper layer. This must be carefully considered during network design, requiring each layer to perform a specific set of well-defined functions.

The combination of layers and protocols is called a network architecture. The specific set of protocols used in a given system is referred to as a protocol stack.
