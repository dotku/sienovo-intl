## 

Segmentation Fault

A [segmentation fault](http://baike.baidu.com/view/120927.htm) refers to accessing [memory](http://baike.baidu.com/view/1082.htm) beyond the memory space allocated to a program by the system. This value is typically stored in the GDTR (Global Descriptor Table Register), which is a 48-bit register. The 32 bits of the GDTR store the base address of the [GDT](http://baike.baidu.com/view/1082.htm) it points to, the next 13 bits store the index corresponding to the GDT, and the last 3 bits include information such as whether the program is in memory, and its privilege level in the [CPU](http://baike.baidu.com/view/2089.htm). The GDT pointed to by the GDTR is a table where each entry is 64 bits. This table stores information such as the starting addresses of the program's [code segment](http://baike.baidu.com/view/1315853.htm) and [data segment](http://baike.baidu.com/view/1005328.htm), along with corresponding segment limits, paging information, program privilege levels, memory granularity, and more.

#### English Introduction

A segmentation fault (often shortened to segfault) is a particular error condition that can occur during the operation of computer software. In short, a segmentation fault occurs when a program attempts to access a memory location that it is not allowed to access, or attempts to access a memory location in a way that is not allowed (e.g., attempts to write to a read-only location, or to overwrite part of the operating system). Systems based on processors like the Motorola 68000 tend to refer to these events as Address or Bus errors.

Segmentation is one approach to memory management and protection in the operating system. It has been superseded by paging for most purposes, but much of the terminology of segmentation is still used, "segmentation fault" being an example. Some operating systems still have segmentation at some logical level although paging is used as the main memory management policy.

On Unix-like operating systems, a process that accesses invalid memory receives the SIGSEGV signal. On Microsoft Windows, a process that accesses invalid memory receives the STATUS\_ACCESS\_VIOLATION exception.

### 2 Common Forms of Segmentation Faults[Edit](http://baike.baidu.com/link?url=q5BY4s6WdLTvMnqZA8yCK-Nel1D8sQhpmfveGuS2ILvgP3cw739Rk5ylvhHDEHkqeGxpQM1kQqwyIAT3Vs8ZjK# "Edit this section")

In [programming](http://baike.baidu.com/view/3281.htm), the following types of practices are prone to causing segmentation [faults](http://baike.baidu.com/view/120927.htm), primarily due to incorrect use of [pointers](http://baike.baidu.com/view/159417.htm).

1) **Accessing system data areas**, especially writing [data](http://baike.baidu.com/view/38752.htm) to system-protected [memory addresses](http://baike.baidu.com/view/404417.htm). The most common case is assigning a [pointer](http://baike.baidu.com/view/159417.htm) to address 0.

2) **Memory out-of-bounds** ([array](http://baike.baidu.com/view/209670.htm) out-of-bounds, inconsistent [variable](http://baike.baidu.com/view/296689.htm) types, etc.): Accessing a memory region that does not belong to your program.

Solution: When writing programs in C/ [C++](http://baike.baidu.com/view/824.htm), most of the [memory management](http://baike.baidu.com/view/4541016.htm) work needs to be done by us. #### Analyzing Core Files

### 3 Several Typical Segmentation Faults[Edit](http://baike.baidu.com/link?url=q5BY4s6WdLTvMnqZA8yCK-Nel1D8sQhpmfveGuS2ILvgP3cw739Rk5ylvhHDEHkqeGxpQM1kQqwyIAT3Vs8ZjK# "Edit this section")

1.  int main(void){ \[1\]  

    char\*s ="hello world";

    \*s ='H';

    }

    When loaded, the system places "hello world" along with other strings and const data into a read-only memory section. During execution, a variable `s` is set to point to the location of this string. When an attempt is made to write to this location, a segmentation fault occurs.

2.  int\*ptr = NULL;

    \*ptr =1;

    This code only creates a null pointer and does not point to a specific memory space. When an assignment is attempted, a segmentation fault occurs.

3.  int main(void){

    main();

    return0;

    }

    Infinite recursion, which leads to a stack overflow, also causes a segmentation fault.
