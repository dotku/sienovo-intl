5. Server (Socket Array): Basically multi-threaded: one thread listens, and after `Accept`, creates a thread to handle the request.

————————————————————————————————————
Ideas:
1: The server has a Listener thread for listening.
2: The server also has a server socket thread for connecting with clients.
3: When a client connection is detected, a server socket thread is created to connect with that client independently.

15. When communicating, the client needs to know the server's IP address. It is the active party.
16. The client requests a temporary port from the server (system), releases it after use, and disconnects. It reconnects when needed. So, when QQ opens a window, is it continuously connected? (It should be. If no window is open, there's no connection. It's just connected to the server.)

17. The server, during communication, is generally the passive party and does not know the client's IP and port information.

18. Use `connect` to bind and connect the client socket to a specific server address.

19. Connect the socket to a remote host (client.c) or local IP (server.c).

20. Receive and send data via socket as required.

21. Port addresses are allocated per process. Sending and receiving must use the same port.

22. Why do processes block when a server handles multiple clients, but threads do not? Each client corresponds to two threads (one for sending, one for receiving); as for the client itself, it only needs two.

23. Request Sending Process
1) Use `gethostbyname()` to obtain host information.
2) Initialize the socket port.
3) Use the `connect` function to send its IP address and other information to the host, waiting for the host to call the `accept` function to accept the request.

24. Host receives request and performs data communication
1) The host uses `accept` to receive the request.
2) Create a child thread and display a welcome message.
3) Receive return information and display connection success.
4) Close the client socket.
5) Close the server socket and terminate the child thread.
