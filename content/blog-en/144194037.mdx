We have implemented input from a Sony camera module and a [Gaode](https://so.csdn.net/so/search?q=%E9%AB%98%E5%BE%B7&spm=1001.2101.3001.7020 "高德") infrared module, along with compression, storage, and RTSP forwarding. Tracking and recognition functionalities have been added, including ByteTrack for multi-target ID assignment. We have also addressed and improved dual-channel transmission and control issues from previous models. If you are interested in using this solution, please contact me. I'm open to discussing it at a reasonable price, offering two options: open-source availability or direct shipment using my proprietary protocol.

![](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/144194037/01_a99ccd5a0b9349559683d521c33cdfda.png)

There are still some issues, such as OSD overlay speed. For example, we lack sufficient training samples to achieve recognition of specific target types. Also, the current tracking [algorithm](https://edu.csdn.net/course/detail/40020?utm_source=glcblog&spm=1001.2101.3001.7020 "算法") is designed for ship targets and hasn't been truly tested in flight for target tracking on a UAV. Some parameters might still require adjustment. Nevertheless, some functionalities have been achieved overall. The basic ideal block diagram is shown below.

![](https://pub-048dcb96257f476697b113fcb5939cb9.r2.dev/blog/144194037/02_4564272eba811ee210f4dbb3ca7937af.png)
